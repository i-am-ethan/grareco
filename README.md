# 使い方

```
# 依頼内容
[ここに説明してほしいトピック名を入れてください] について、初心者にも分かりやすく教えてください。

# 出力形式とスタイル
以下の要件を満たす「グラレコ風」の解説を、**単一のHTMLファイル**で生成してください。

## デザイン・レイアウト要件：
1.  **全体的なスタイル**: グラフィックレコーディング（グラレコ）のように、視覚的に魅力的で、読んでいて楽しくなるようなデザインを目指してください。
2.  **背景色**: ページ全体の背景は薄い青色 (`#e0f7fa` やそれに類する色) に設定してください。
3.  **コンテンツエリア**: 主要なコンテンツ部分は、白い背景に角丸、影などをつけて、カードのように見やすくしてください。
4.  **タイトル・見出し**: メインタイトルやセクション見出し (`h1`, `h2`など) は、視覚的に目立つように、大きめのフォントサイズ、アクセントカラー、下線、背景色、またはアイコンなどを効果的に使用して装飾してください。ポップなフォントの使用も検討してください。
5.  **図解・チャート**: 説明を補完するために、HTMLとCSS（必要であれば簡単なSVG）を使って、シンプルな図やフローチャート、コンポーネントの構成図などを適宜挿入してください。点線やボックス、矢印などを活用してグラレコ感を表現してください。
6.  **画像**: 必要に応じて、説明の助けになる画像（フリー素材など、著作権に配慮したもの）を挿入してください（必須ではありません）。
7.  **コード表示**: コード例を含める場合は、`<pre><code>`タグを使用し、背景色をつけるなどして見やすく整形してください。

## コンテンツ要件：
1.  **言語**: 日本語で記述してください。
2.  **可読性**: 専門用語は避け、平易な言葉で、箇条書きや短い段落を使い、非常に分かりやすく説明してください。重要なキーワードは太字や色などで強調してください。
3.  **構成**: トピックの概要、主な特徴や利点、基本的な仕組みやコンセプト、始め方（もしあれば）など、論理的な流れでセクション分けしてください。

## インタラクティブ要素：
1.  **アニメーション**:
    * ページ読み込み時やスクロール時に、セクションや図解などが**フェードイン**したり、**スライドイン**したりするような、控えめで心地よいJavaScriptアニメーションを導入してください (例: Intersection Observer APIの使用)。
    * タイトルなど、特に目立たせたい要素に簡単なアニメーション（例: バウンドイン）を追加しても良いです。
2.  **復習クイズ**:
    * 解説の最後に、内容の理解度を確認するための簡単な**3択または4択のクイズを3問**程度設置してください。
    * ユーザーが回答を選択し、「答え合わせ」ボタンなどをクリックしたら、JavaScriptを使って**各問題の正解・不正解を表示**してください。
    * 正解の場合は称賛のメッセージと簡単な解説、不正解の場合は正解の選択肢と簡単な解説を表示するようにしてください。
    * フィードバックの表示/非表示、スタイル変更もJavaScriptで制御してください。

# その他
* HTML、CSS、JavaScriptはすべて単一のHTMLファイル内に記述してください（インラインまたは`<style>`, `<script>`タグ内）。
```
